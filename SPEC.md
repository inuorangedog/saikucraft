# SaikuCraft 仕様書

## サービス概要
- **名称**: SaikuCraft（サイククラフト）
- **コンセプト**: 手描きクリエイターと安心して繋がれる同人特化コミッションサービス
- **キャッチコピー**: 「あなたの世界を、届ける人がいる。」
- **ターゲット依頼側**: 表紙・ロゴ・お品書き等を外注したい同人作家
- **ターゲット受託側**: AIを使わず手で描く・作ることに誇りを持つクリエイター

## 技術スタック
- Next.js 15 (App Router)
- Supabase (認証・DB・Storage)
- Stripe Connect Express (決済)
- Cloudflare R2 (画像ストレージ)
- Tailwind CSS
- TypeScript
- Vercel (ホスティング)

## 募集案件一覧
- カード表示：参考画像・タイトル・予算（1人あたり）・募集人数・希望納期・タグ・イベントタグ・募集締め切り
- 並び順：新着順・締め切りが近い順・予算が高い順・イベント別フィルター
- 取引パターン：募集型（MVP）・指名型（MVP）・提案型（v2）

## 取引画面
### レイアウト
- 左側：取引情報・進捗ステータス・アクションボタン
- 右側：メッセージ（画像添付可・テキスト）

### 進捗ステータス（自動更新）
1. 取引開始（仮払い完了）
2. ラフ提出待ち
3. ラフ確認中（クリエイターがラフを提出）
4. 詳細ラフ確認中
5. 着手済み・制作中（依頼者が詳細ラフを承認）
6. 納品済み・確認中
7. 完了（依頼者が承認 or 納品後3日で自動承認）

### アクションボタン
- クリエイター：ラフを提出する・詳細ラフを提出する・修正版を提出する・納品する
- 依頼者：ラフを承認する・修正を依頼する・納品を承認する・追加料金を承認する・異議申し立て

### ファイル添付
- 納品ボタン：ZIP・PSD・PNG等 最大1GB・30日後自動削除
- メッセージ添付：画像のみ・3枚・1枚5MBまで・30日後自動削除
- 大容量：ギガファイル便等の外部サービスを案内

### 修正回数カウンター
- 取引画面内に「ラフ修正：2/3回」と残り回数を表示
- 上限に近づいたら「追加料金が発生する可能性があります」と自動案内

## エスクロー（保護決済）
- 着手の定義：依頼者が詳細ラフを承認した時点
- 着手前キャンセル：全額返金（運営負担）
- 着手後キャンセル：返金なし
- 締め切り3日超過：自動で全額返金（クリエイター負担）
- 納品後3日で自動承認・送金
- 締め切り超過→自動返金が3回でアカウント強制停止
- 部分納品トラブル：異議申し立て→3日間交渉→運営が証拠ベースで仲裁

## BOOSTシステム
- 取引完了後のみ送れる
- 上限：取引額×3倍（最大¥50,000）
- 金額単位：100円単位・最低¥500
- 手数料：7%
- 直近30日以内にBOOSTされたクリエイターに🖌️人気バッジ表示

## マイページ
### モード切り替え
- 1アカウントでクリエイター・依頼者両方の役割
- 完全分離：クリエイターモード（暖色）/ 依頼者モード（寒色）

- プロフィール編集・受注中の取引一覧（未読バッジ）

### 依頼者モード

- 募集ページの管理・お気に入りクリエイター

- サイト上でAI使用禁止を明言
- 登録時に「人間が制作した」申告を求める
- 通報ポイントが一定数を超えたクリエイターにはPSDデータまたは制作動画の提出を義務付け
- 提出できない場合はアカウント停止

## 指名型取引フロー

```
1. 依頼者がクリエイターページから「依頼する」ボタン
   → 依頼内容・予算・納期・参考画像を入力して送信

2. クリエイターが確認（3日以内に返答必須・無返答は自動辞退）
   → 承諾 / 辞退 / 条件変更を提案

3. 条件変更の場合
   → 依頼者に通知
   → 依頼者が承諾 / 再交渉 / キャンセル
   → 交渉は何往復でもOK（専用チャット画面）

4. お互いが条件に合意
   → 依頼者が仮払い → 制作開始（以降は募集型と同じフロー）
```

### 交渉メッセージで変更できる項目
- クリエイター側：予算・納期・メッセージ（自由記述）
- 依頼者側：予算・納期・依頼内容の補足

### negotiationsテーブル
- id
- creator_id
- client_id
- title（依頼タイトル）
- description（依頼内容）
- budget（依頼者の希望予算）
- deadline（依頼者の希望納期）
- reference_images（参考画像）
- status（交渉中/合意済み/辞退/キャンセル）
- created_at

### negotiation_messagesテーブル
- id
- negotiation_id
- sender_id
- proposed_budget（提案予算・任意）
- proposed_deadline（提案納期・任意）
- message（自由記述・必須）
- created_at

### 交渉→取引のデータ引き継ぎ
- 仮払い完了時にnegotiationsのデータを自動でtransactionsに生成
- transactionsテーブルにnegotiation_idを追加（指名型の場合のみ・NULLあり）
- transactionsテーブルのlisting_idは募集型の場合のみ（NULLあり）

## 退会処理

## Stripe Webhook＋Vercel Cron Jobs

## 技術的決定事項

### 決済方式
```
サービス：Stripe
方式：Separate charges and transfers
理由：段階的キャンセルポリシー（全額/50%/返金なし）の実装に必須
　　　依頼者→SaikuCraftのStripe口座に仮払い
　　　取引完了後にcreate_transferでクリエイターに送金
　　　キャンセル時はrefundで返金
法的対処：利用規約に決済代行スキームとして明記・SKIMAと同じ構造
```

### メッセージのリアルタイム更新
```
Supabase Realtimeを使用
追加コストなし・LINEのように即時表示
取引画面のメッセージに適用
```

### ファイルアップロード
```
R2 Presigned URLを使ってブラウザからR2に直接アップロード
理由：Next.js経由だとVercelのタイムアウト（10秒）と
　　　ボディサイズ制限（4.5MB）に引っかかる
　　　納品ファイルが最大1GBあるため必須
```

### Supabase RLS（行レベルセキュリティ）
```
実装前に必ず設定する
主なルール：
・自分の取引のメッセージしか見えない
・自分のプロフィールしか編集できない
・adminロールは全データにアクセス可能
・他人のStripe情報は見えない
```

### お気に入り通知のバッチ処理
```
MVP：一斉送信（取引数が少ないため問題なし）
スケール後：キューに入れて分散送信に移行
```

### ページネーション

```
方式：ページ番号（Google型）
```

## 利用規約 骨子

### 第1条 定義
```
・本サービス：SaikuCraft（運営者：個人事業主）

・取引：依頼者とクリエイター間の制作委託契約
・仮払い：取引開始時に依頼者が支払う代金
```

### 第2条 利用条件
```
・18歳以上であること（自己申告）
・日本国内在住であること（MVP段階）
・Googleアカウントを保有していること
・本規約に同意したうえで登録すること
```

### 第3条 禁止事項
```
・AI生成物の納品・出品
・他者の著作物の無断使用
・未成年者への性的コンテンツの提供
・虚偽のプロフィール・実績の掲載
・サービス外での直接取引（取引成立後の誘導）
・スパム・嫌がらせ行為
・法令に違反する行為
⚠️ サービス外直接取引の禁止は重要（手数料回避防止）
```

### 第4条 決済・エスクロー
```
・依頼者は取引開始時に仮払いを行う
・仮払いはStripeを通じて行う
・依頼者が当社に支払いを完了した時点で
　クリエイターへの支払い義務が履行されたものとみなす
・当社は取引完了後にクリエイターへ代金を送金する
・送金時に所定の手数料（7%）を控除する
⚠️ 決済代行スキームの文言は弁護士確認推奨
```

### 第5条 キャンセルポリシー
```
・取引開始〜ラフ提出前：全額返金
・ラフ提出後〜詳細ラフ承認前：50%返金
・詳細ラフ承認後：返金なし
・締め切り3日超過：全額返金（クリエイター負担）
・返金はStripeを通じて行い数日かかる場合がある
```

### 第6条 著作権
```
・納品物の著作権はクリエイターに帰属する
・依頼者は取引完了時に個人的な使用権を取得する

　取引開始時にシステム上で申告すること
・上記の特別条件は追加料金が発生する場合があり
　クリエイターは条件に同意しない権利を持つ
・合意した条件はシステム上に記録される
・取引完了後に追加の利用が発生した場合は
　クリエイターと直接合意すること
・当社は取引完了後の二次利用に関するトラブルについて
　一切の責任を負わない
```

```
・R18コンテンツの取引は18歳以上のユーザー間のみ許可
```

## 取引開始時の著作権・利用範囲設定

### 依頼者が選択する項目（デフォルトすべてOFF）
```
□ 著作権の譲渡を希望する
  ※追加料金が発生する場合があります
  ※クリエイターが承諾しない場合があります

□ ポートフォリオへの掲載を禁止する
  ※追加料金が発生する場合があります

□ 商業利用を含む
  ※クリエイターが「商業利用可」設定の場合のみ選択可能
  ※追加料金が発生する場合があります
```

### システムの動き
```
どれかをONにした場合：
→ 警告表示「この設定は追加料金が発生する可能性があります」

クリエイターへの通知：
→「この依頼には特別条件が含まれています。
　 追加料金を請求する場合は交渉画面で提示してください」

クリエイターは：
→ 追加料金を提示して交渉できる
→ 条件が合わない場合は辞退できる
```

### DBへの追加（transactionsテーブル）
```
- wants_copyright_transfer（著作権譲渡希望・デフォルト：false）
- wants_portfolio_ban（掲載禁止希望・デフォルト：false）
- wants_commercial_use（商業利用希望・デフォルト：false）
```
